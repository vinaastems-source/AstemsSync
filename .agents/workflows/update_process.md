# AstemsSync 업데이트 워크플로우 (Update Workflow)

사용자 실행부터 업데이트 완료 및 시스템 갱신까지의 전체 비즈니스 로직을 정의합니다.

## 1. 초기화 및 로그인 단계
* 1단계: 사용자가 AstemsSync.exe 실행 및 FF 창 노출
* 2단계: FB에서 로컬 `Astems.ini` 파일 파싱 (매장 정보, 현재 버전 추출)
* 3단계: 사용자 ID 및 Password 입력 후 로그인 요청

## 2. 검증 및 정책 결정 단계
* 4단계: FB가 BF 서버로 로그인 정보 및 로컬 포스 환경 정보 전송
* 5단계: BF에서 유효한 사용자인지 검증
* 6단계: BF에서 해당 매장/포스의 업데이트 대상 여부 판단
    * 강제 업데이트(Forced) 여부 체크
    * 점진 배포(Canary) 대상 여부 체크
    * 버전 고정(Pinning) 여부 체크
* 7단계: BF에서 업데이트 패키지 정보(URL, 해시 등)를 FB로 응답

## 3. 업데이트 수행 단계
* 8단계: FB에서 파일 다운로드 시작
    * 네트워크 장애 시 재시도 로직 작동
    * 진행 상황(Progress)을 FF로 실시간 전달 (Progress Bar 갱신)
* 9단계: 다운로드 완료 후 파일 해시 검증
* 10단계: FB에서 다운로드된 패키지 압축 해제
* 11단계: 지정된 폴더로 파일 교체 및 적용

## 4. 완료 및 갱신 단계
* 12단계: FB에서 로컬 `Astems.ini` 파일의 버전 정보를 신규 버전으로 업데이트
* 13단계: BF 서버로 최종 처리 결과(성공 알림) 보고
* 14단계: FF에서 완료 메시지 표시 및 필요 시 프로그램 재실행 안내
