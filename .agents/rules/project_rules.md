# 프로젝트 개발 표준 및 규칙 (Project Rules)

AstemsSync 프로젝트의 기술적 타안성과 안정성 확보를 위한 규칙입니다.

## 1. 기술 스택 기반 원칙
* UI (FF): SvelteKit 기반의 반응형 디자인을 유지하며, 사용자에게 명확한 상태 피드백을 제공한다.
* 로직 (FB): Rust를 사용하여 메모리 안전성을 확보하고, 파일 시스템 접근 및 네트워크 통신 시 예외 처리를 엄격히 한다.
* 서버 (BF): Kotlin으로 작성하며, DB 트랜잭션의 정합성을 보장하고 대량의 클라이언트 요청에 대응 가능한 구조를 설계한다.

## 2. 업데이트 안정성 규칙
* 원자성(Atomicity): 파일 교체 작업은 전체가 성공하거나, 실패 시 원래 상태를 유지할 수 있도록 설계한다.
* 무결성 검증: 다운로드된 모든 패키지는 해시(Hash) 검증을 거친 후 적용한다.
* 재시도 전략: 네트워크 실패 시 즉시 종료하지 않고 지수 백오프(Exponential Backoff) 알고리즘을 활용해 재시도한다.

## 3. 문서화 및 로깅
* 모든 기여는 `.agents` 폴더 내의 설계 명세(specs)와 워크플로우(workflows)에 기반한다.
* 로깅 시 타임스탬프, 매장정보, POS번호를 포함하여 문제 발생 시 추적이 가능하게 한다.
* 하이픈(-) 사용을 지양하고 명확한 문장을 사용한다.
